# 验证清单

以下命令按当前仓库的实际脚本和工作流整理，用于验证当前可执行基线、两个前端构建链路、Docker 编排，以及 staging 部署后的基础可用性。

## 后端当前基线

```bash
cd vocata-server
mvn -Dmaven.repo.local=/tmp/juhao_m2repo test
```

说明：当前后端硬门槛是一组稳定的 smoke test，用于确认最基础的 Spring MVC 接口和测试链路可用。

## 后端构建基线

```bash
cd vocata-server
mvn -Dmaven.repo.local=/tmp/juhao_m2repo -Dmaven.test.skip=true package
```

说明：构建基线继续保留，用于确认 Maven 依赖、编译和打包链路可用。

## 用户端

```bash
cd vocata-web
npx eslint .
npm run type-check
npm run build
```

## 管理端

```bash
cd vocata-admin
npx eslint .
npm run type-check
npm run build
```

## Docker 开发环境

```bash
docker compose config -q
docker compose ps
```

## Staging 部署后检查

```bash
cd /home/deploy/deploy/vocata/repo
docker compose --env-file /home/deploy/deploy/vocata/.env ps
curl -fsS http://127.0.0.1:9009/api/health
curl -fsS "http://127.0.0.1:9009/api/open/character/list?pageNum=1&pageSize=2"
```

说明：以上命令在 staging 主机上执行，用于确认部署后的容器状态和后端接口可用性。
